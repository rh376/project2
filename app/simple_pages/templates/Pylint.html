Pylint

A tool that checks for errors in Python code, tries to enforce a coding standard and looks for code smells.

Pylint is defined as a Python Static code analysis tool for analytical purposes. It can scan for programming errors, implement best practises in coding standards, identify code violations and provide refactoring suggestions. It is fully user customizable and has integral special functions to control individual errors and alerts within the user’s code. It gives the user flexibility to write their own custom plugin and executable from an extensive configuration file. Pylint will display a series of messages as it does code analysis and the code is given an overall mark based on number and severity of warnings and errors. Pylint was developed in 2003 at Logilab and funded Sylvain Thénault to lead its development and until the 2015 project went under Python Code Quality Authority umbrella.
