Git Cheat Sheet

Git is a DevOps tool used for source code management. It is a free and open source version control system used to handle small to very large projects efficiently.
Git is used to tracking chances in the source code, allowing multiple developers to work together at the same time.



  • git branch - Lets you create, list, rename, and delete branches
  • git merge - Git's way of putting a forked history back together again
  • git commit - Captures a snapshot of the project's currently staged changes
  • git tags - Ref's that point to specific points in Git history
  • git repositories - A collection of files of various different versions of the project

More Git Commands:

  • git add - Adds a change in the working directory to the staging area
  • git init - Creates a new local repository
  • git push (origin master) - Sends changes to the master branch of your repository
  • git status - Lists the files you've changed and those you still need to add or commit
  • git pull - Fetch and merges changes on the remote server to your working directory
  • git grep - Searches the working directory for foo():
