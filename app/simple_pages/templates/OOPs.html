OOP definitions -

Object-oriented programming is a programming paradigm based on the concept of "objects", which can contain data and code - data in the form of fields, and code, in the form of procedures. One feature of objects is that its own procedures can access and often modify the data fields.

  • instantiation - the creation of a real instance or particular realization of an abstraction or template such as a class of objects or a computer process.
  • class - instruction for creating objects giving values and behavioral implementations
  • object - data created by developer that can be structure, method, function, or variable
  • namespace - names used to identify different kinds of objects
  • constructor - method that creates a special object of a class
  • fixture - sample data
  • type hint - solution that indicates value type
  • type cast - changing object type
  • unit test - testing small piece of code
  • static - another way to say constant
  • instance method - method that requires an object of its class to be created first
  • instance property - used to store state of object
  • static method - method that belongs to a class
  • static property - used to refer to common property
  • encapsulation - grouping of data including methods to one object
  • inheritance - mechanism of basing an object or class upon another object
  • polymorphism - use of a single symbol to represent multiple different types
  • abstraction - removing physical, spatial, or temporal details or attributes in the study of objects or systems
