OOP

Object Oriented Programming

Object-oriented programming is a computer programming model that organizes software design around data, or objects, rather than functions and logic. OOP language allows to break the program into the bit-sized problems that can be solved easily.

Definitions:

  • Instantiation - To create an instance by defining one particular variation of object within a class.
  • Class - A blueprint for creating objects, providing initial values for state and implementations of behavior.
  • Object - Units of code that are eventually derive from the process.
  • Namespace - A declarative region that provides a scope to the identifiers inside it.
  • Constructor - A special method of class or structure in OOP that initializes a newly created object.
  • Fixture - Sample Date.
  • Type Hint - A formal solution to indicate the type of value within your code.
  • Type Cast - A way of changing an object from one data type to the next.
  • Unit Test - Programs that test classes.
  • Static - The variable or method marked as such is available at the class level.
  • Instance Method - A method that belongs to an instance of a class.
  • Instance Property - Applies to specific system servers.
  • Static Method - A method that belongs to a class rather an instance of a class.
  • Static Property - Accessed using the Scope Resolution Operator and cannot be accessed through the object operator.
  • Encapsulation - The grouping of data with the methods that operate on the data.
  • Inheritance - The procedure in which one class inherits the attributes and methods of another class.
  • Polymorphism - Situations in which something occurs in several forms.
  • Abstraction - Hides the internal implementation details.

SOLID is a set of design principles that are used in OOP software development. SOLID principles include Single Responsibility Principle, Open-Closed Principle, Liskov Substitution Principle, Interface Segregation Principle, and Dependency Inversion Principle.

A design pattern provides a general reusable solution for the common problems that occur in software design.
