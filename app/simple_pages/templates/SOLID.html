SOLID

The first five object-oriented design principles

SRP

Single Responsibility Principle: A class should have one and only one reason to change, meaning that a class should have only job.

OCP

Open-Closed Principle: Objects or entities should be open for extension but closed for modification. This means that a class should be extendable without modifying the class itself.

LSP

Liskov substitution principle: Every subclass or derived class should be substitutable for their base or parent class.

ISP

Interface Segregation principles tell us that a class should only have the interface needed and avoid methods that wonâ€™t work or that have no reason to be part of that class.

DIP

Dependency Inversion principle helps abstractions to not depend on details. Details should depend on abstraction. High-level modules should not depend on low-level modules. Both modules should depend on abstractions.
