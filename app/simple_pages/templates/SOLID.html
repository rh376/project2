SOLID

Basic learning steps for every code developer

SRP

Single Responsibility Principle - the emphasis of every component of your code should have one, and only one responsibility. Furthermore, the only caveat to implementing SRP is making sure you have improperly assigned responsibilities from various areas in code.

OCP

The O stands for open closed principle which states you should not need to modify the code you have already written to accommodate new functionality, but to just add to it.

LSP

Derived classes must be substitutable for their base classes, also know as Liskov substitution principle.

ISP

Interface Segregation principles tell us that a class should only have the interface needed and avoid methods that wonâ€™t work or that have no reason to be part of that class.

DIP

Dependency Inversion principle helps abstractions to not depend on details. Details should depend on abstraction. High-level modules should not depend on low-level modules. Both modules should depend on abstractions.
